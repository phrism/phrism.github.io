download to doodleGrive-cli to my machine an

There's some restrictions when doing sql injection, for example the sanitize_string function will limit your input with 39 character, and the spectial character like ";","/","|" will be replaced with "". However, I can still bypass these restrictions and get rce with load_extension and evil-so file.
With the sqlite injection, we can input

"+load_extension("./e")--

into the username and load an .so file, to bypass "/", we use char() to encode our exploit:

"+load_extension(char(46,47,101))--

which is 35 characters.
